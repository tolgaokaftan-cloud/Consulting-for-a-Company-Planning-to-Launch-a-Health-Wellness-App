Consulting for a Company Planning to Launch a Health & Wellness App¶

1. Introduction

Since we are in 2026, all of us know that massive use of smartphones/tablets/laptops has been identified as one of the major issues that is impacting individuals' mental health and productivity. Also they are having lack of sleep and it affects their daily routines. Our collaborating firm is a healthy lifestyle application that is intended to improve individuals' lifestyle and influence them to use smartphones in a healthy way. However, currently, the application is unable to identify and evaluate the level of addiction of its users. This is where we are.

The major aim of this project is to create a predictive model that can forecast individuals' levels of smartphone addiction based on their characteristics. By being able to identify and forecast individuals' levels of smartphone addiction, the firm can improve its application's recommendation and value proposition in the competitive market.

The data we collect from that firm will help us to improve their business. The main goal is here to detect and automate the customers behaviors with that firm can adjust their recommendation settings for people accurately.

1.1 The Data We Got and How We Use

The dataset they gave represents behavioral and usage patterns of 7,500 smartphone users. It includes different aspects of their behavior. Each row in the dataset represents how users behave and interact with their smartphone. It includes usage time, application preferences, and frequency of checking their smartphone and its potential impact on sleep patterns.

The dataset does not contain individual numerical values. It provides a comprehensive behavioral profile for users. It also reveals that smartphone addiction is a multidimensional issue that depends on a set of different factors, including time spent using the smartphone, the nature of the applications used by the users, and patterns of behavioral repetition.

The target variable in this dataset is the addiction level. It helps to classify users according to different levels of addiction. This helps to create a supervised machine learning problem. This dataset helps to identify patterns between user behavior and addiction levels. It can be used to develop machine learning model that we are gonna do.

1.2 How to Collect Relevant Data and Formulate as a Machine Learning Problem

The relevant data for this problem can be collected through a combination of behavioral data and self reported information through the mobile app. The app could track the behavior of the users with different metrics like the time spent on the screen each day, the number of times using the phone, the type of apps the user mostly uses (for example, social media, entertainment), and the time spent on the app.

The app can also collect self reported data like the quality of the user's sleep and the user's mood through short surveys. For sure, the data should be anonymous, and the users should give their consent to the data being collected. The combination of both types of data will give us a better understanding of the behavior of the users.

Our business case can be viewed as a supervised learning problem of classification type, in which the task is to make predictions on the level of smartphone addiction of the user based on the provided behavioral characteristics. In this regard, the input variables are the characteristics of the user behavior, which are represented by different indicators of the level of smartphone addiction, whereas the output variable is the level of addiction.

The model is trained on the data provided and then can make predictions on the level of addiction of the user behavior, thus allowing the application to make recommendations on the user behavior.

1.3 Data Quality Issues

During the initial exploration of the data, some issues can be identified. For instance, there are missing values for some data points in the target variable which is “addiction_level.” This needs to be addressed properly by removing the relevant rows of data. Furthermore, there are categorical and numerical variables in the data set that need to be preprocessed for sure. Lastly, there are no major problems concerning data consistency and structure. This means that the data is properly structured for analysis.

In the target variable there are different classes of "smartphone addiction level" for example, Mild, Moderate, and Severe. However, it is essential to check if the classes are balanced. If one class dominates the others, then the data set is considered imbalanced. This can affect the performance of the model. In this case, oversampling and undersampling techniques can be used to balance the data set. If the classes are balanced, then there is no need to apply any sampling techniques.

Since the machine learning algorithm is learning the pattern of the data, it is important to check the performance of the data. Accuracy could not the best way to measure the performance of the data set. In the case of imbalanced data sets, other metrics like precision, recall, and F1 score should also be considered.

Hyperparameter tuning was used for the Random Forest model to optimize its performance. Hyperparameters like the number of trees, depth of trees, and feature selection method were tuned to optimize bias-variance trade-off and improve the model's ability to generalize.

Why we choose n = 100, 200 depth = 10,20 min samples = 2,5 because: The values used for the hyperparameters were chosen to ensure that different levels of complexity are tested without making the process too slow. Since n_estimators are commonly used values that yield stable results. The max_depth values were chosen to ensure that the complexity of the trees, both shallow and deeper, are used to prevent overfitting. The min_samples_split values are used to find a balance between a flexible model and a more general one. Lastly, the max_features options chosen, “sqrt” and “log2,” are commonly used to enhance the performance of the models by introducing randomness.

Unlike the Random Forest model, in the Gradient Boosting model, the trees are constructed one after the other, in which the focus is on learning the errors of the previous model. This helps the model in learning complex and overlapping classes, like the “Moderate” and “Severe” classes of addiction in the data set.

3. Conclusion

If we conclude, multiple models were created and tested. Gradient Boosting was seen to be the best model due to results. In addition to that, the macro F1 score was used to ensure that all classes were balanced in the evaluation.

LIMITATIONS:

The main limitation was that the classes “Moderate” and “Severe” were overlapping. We were struggling to find a way to improve scores. This made it hard to classify. In addition to that, the dataset lacked distinctive features. This reduced the accuracy of the model. In the end, the accuracy was moderate.

IMPLICATION FOR BUSINESS:

The model was classifying the users who were likely to have higher levels of addiction. This helped the company to be proactive in ensuring that the users were helped.

For better performance, the company can use detailed data and feature engineering techniques. Moreover, users can be grouped based on risk factors and take appropriate actions for them. The model can be further improved with new data.

The most important features are associated with user activities such as screen time, social media usage, opening applications, and notifications. These features are directly related to user activities and can be used to show levels of addiction. Gradient Boosting is a moderately explainable model. Although it is not as easy to explain as Decision Trees, feature importance can be used to gain some knowledge about the features that are affecting the prediction.
